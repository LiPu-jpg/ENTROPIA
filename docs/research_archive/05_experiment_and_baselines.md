## 消融矩阵

| 变体 | 禁用组件 | 问题 |
| --- | --- | --- |
| 无 Need | `N_t = 1` | 基于不确定性的监督需求是否有意义？ |
| 无 Utility | `U^k_t = 1` | 信号特定的进展估计是否必要？ |
| 无 Reliability | `L^k_s = 1` | 结果对齐校准是否有意义？ |
| 无 Risk | `H^risk_t = 0` | 作弊抑制是否有意义？ |
| 无预算 | `B_s = constant large` | 稠密预算控制是否有意义？ |
| 仅单一信号 | 一次一个 `S^k` | 哪些过程信号具有真实价值？ |

## 主要任务

### 搜索 QA

推荐用于首次方法验证。

示例：

- HotpotQA 风格的多跳搜索；
- 带检索的 Natural Questions；
- Search-R1 风格环境。

原因：

自然暴露信息增益、冗余检索、证据覆盖和最终答案正确性。

### 工具/客服任务

适用于第二阶段验证。

示例：

- tau-bench 或 tau2-bench 风格的工具调用；
- SIMIA / AReaL / fuvty 合成轨迹。

原因：

强调可解析性、工具有效性、多轮控制和格式作弊。

## 指标

| 类别 | 指标 |
| --- | --- |
| 最终成功率 | EM、F1、通过率、任务完成率 |
| 效率 | 工具调用次数、token 数、延迟、重复搜索次数 |
| 奖励健康度 | 奖励-成功相关性、过程-结果一致性 |
| 门控质量 | 门控精度、激活率、稠密预算使用量 |
| 作弊控制 | 重复率、引用操纵、仅格式成功、发散 |
| 稳定性 | 跨随机种子方差、崩溃步数、KL 漂移 |

## 预期声明

ENTROPIA-v2 不仅应提升最终成功率，还应展示：

1. 比固定稠密奖励更低的奖励-成功发散；
2. 比始终稠密的过程奖励更好的搜索效率；
3. 比固定 CRAFT 奖励更稳定的训练；
4. 比仅熵门控更好的门控精度；
5. 在稠密奖励可用时更低的作弊率。

## 需包含的失败分析

所需失败案例：

1. 门控对无用新颖性打开。
2. 门控对高置信错误行为关闭。
3. Reliability 抑制了先前有用的信号。
4. Risk 抑制了格式/引用/重复作弊。
5. 稠密奖励上升而最终成功率下降。

这些案例之所以重要，是因为它们使路由叙事可视化，而非仅仅是数值上的。
